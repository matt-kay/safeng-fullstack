# SafeMe Android Setup Specification

## Overview
This document outlines the Android-specific project setup and environment configuration for the SafeMe mobile application.

## Core Requirements
1. **Target Platform**: Android (Min SDK 24+, Target SDK 34+)
2. **Language**: Kotlin 1.9+
3. **UI Framework**: Jetpack Compose
4. **Architecture**: MVVM / Clean Architecture with Feature modules or package grouping (e.g., `feature/auth`, `feature/home`, `core/network`).
5. **State Management**: `ViewModel` with `StateFlow`.
6. **Routing**: Jetpack Navigation Compose.
7. **Dependency Manager**: Gradle (Kotlin DSL preferred).

## Setup Steps
1. **Initialize Android Studio Project**:
   - Create a new "Empty Activity" Compose project.
   - Application ID: `com.scitylana.safeme`.
   - Build Configuration Language: Kotlin DSL.
2. **Environment Configuration (Flavors)**:
   - Configure `productFlavors` in `app/build.gradle.kts` for `dev`, `staging`, and `prod`.
   - Setup separate `applicationIdSuffix` and API base URLs per flavor via `BuildConfig`.
3. **Add Dependencies**:
   - Add Firebase BoM to `libs.versions.toml` or `build.gradle.kts`. Include Auth, Crashlytics, and Analytics.
   - Add Retrofit, OkHttp, KotlinX Serialization (or Gson/Moshi).
   - Add Hilt (if using Dependency Injection).
4. **Theme Configuration**:
   - Modify `Color.kt` and `Theme.kt` generated by Compose emphasizing `#014342` and `#74E7B2`.
   - Implement `darkTheme` mapping for the `MaterialTheme`.
5. **Firebase Initialization**:
   - Add the Google Services Gradle plugin.
   - Place respective `google-services.json` files in the src flavor directories (`src/dev/`, `src/prod/`).
6. **Deep Linking Setup**:
   - Define intent filters with `android:autoVerify="true"` in the `AndroidManifest.xml`.
   - Handle incoming nav arguments in the Compose Navigation graph.
